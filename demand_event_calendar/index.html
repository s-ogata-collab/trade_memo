■ 指標の見方 — LEGEND

影響度
市場全体へのインパクトの大きさ。特大＝指数レベル／高＝テーマ・セクター／中＝限定的。大きい＝儲かる、ではない（取りやすさと別軸）。
取りやすさ
個人が実際にエッジ化しやすいか。★★★＝相手・期限・対象が明確で後追いでも間に合う／★＝影響大でも結果が二方向・織り込みが速く直前に当てに行く価値がない。迷ったら★★★だけやる。
混雑
どれだけ多くの参加者が見ているか。高＝機関・裁定勢が徹底分析済み→先回り禁止・通過後のみ・実施日引けピークアウト警戒。
型 A/B/C/D
A=Structural Event／B=Behavioral Reversal／C=Theme Propagation／D=Defensive Rotation。A→B=イベントはAだが取り方は通過後のB。
trade_bias
実際に取るトレードの方向（需給方向とは別軸）。long_post=通過後の買い／long_pre=事前の買い／reversal=反転狙い／watch=観察のみ・新規建てしない／two_sided=発表内容次第。
size / 保有
推奨ポジションサイズ（normal/small/very_small/watch_only）と想定保有期間・手仕舞いトリガー(exit)。Structural Edgeは期限付きなので伸びなければ期間内に降りる。
事前/当日/通過後
フェーズ別行動。営業日ベースで今日が該当するフェーズを緑枠ハイライト（祝日・休場を除外し前後3営業日で判定）。
✓ Go条件
やってよい確認条件。これが揃って初めて検討可。テーマモニターのTOPIX相対・Breadth・出来高で確認する項目。
✕ 無効化
「イベントはあるが今回は効いていない」の判定。1つでも成立なら見送り。「イベント＝必ず動く」という錯覚への防波堤。
事後ログ
各カード下部の「事後検証ログ」を開いて結果を記録（worked/failed/neutral、無効化条件の発動有無、効いたテーマ、メモ）。ブラウザに保存され、来期の精度を上げる。JSON書き出し可。
ID / SOURCE
イベント固定ID（検証ログ紐付け用）と情報の出所・確認日。「要直前確認」印は推計金額・未確定日程。トレード前に一次情報で更新。
毎朝の4ステップ：①今日どのイベントのどのフェーズか（上部パネル）→ ②対象テーマがTOPIXに勝ちBreadth・出来高が伴うか（Go条件／テーマモニター）→ ③損切りが近い価格はどこか → ④相手は誰でいつ需給が終わるか。4つ揃わなければ見送り。
